# 固件说明(firmware/)

本目录下的每个文件夹都是一个**独立、可单独烧录**的 Arduino 工程,对应 `GUIDE.md` 里的搭建阶段。
用 Arduino IDE 打开对应 `.ino` 文件即可,开发板选 **ESP32 Dev Module**。

## 四个阶段

| 文件夹 | 对应阶段 | 功能 |
|---|---|---|
| `stage1_read_uid/` | 阶段1 | 只接 RC522,刷卡后串口打印卡号(UID) |
| `stage2_whitelist_led/` | 阶段2 | 阶段1 + 白名单判断,认识的卡绿灯、陌生卡红灯 |
| `stage3_servo_lock/` | 阶段3 | 阶段2 + 舵机开锁:认识的卡开锁3秒后自动锁上 |
| `stage4_full_core/` | 阶段4(整合) | 阶段1~4 全部整合:开锁 / 红灯+蜂鸣报警 |
| `stage5_smart/` | 阶段5(智能升级) | 阶段4 + WiFi/NTP对时 + 白名单持久化 + 主卡录卡 + 防暴力破解 + 网页看板,**推荐最终使用这个** |

## 依赖库(Arduino IDE 库管理器安装)
- **MFRC522**(作者 GithubCommunity)
- **ESP32Servo**

## 每个阶段需要的接线

- **阶段1**:仅 RC522(SS=5, SCK=18, MISO=19, MOSI=23, RST=27, VCC=3.3V, GND=GND)
- **阶段2**:阶段1 + 绿LED(GPIO32,经220Ω)+ 红LED(GPIO33,经220Ω)
- **阶段3**:阶段2 + 舵机(信号GPIO13,VCC接5V/VIN,GND共地)
- **阶段4**:阶段3 + 有源蜂鸣器(GPIO25)

完整接线表见项目根目录 `GUIDE.md` 第3节。

## 烧录顺序(建议按顺序做,每步先跑通再进行下一步)

1. 烧 `stage1_read_uid`,打开串口监视器(115200),刷卡记下打印出来的卡号。
2. 把卡号填入 `stage2_whitelist_led` 的 `WHITELIST` 数组,烧录测试绿/红灯。
3. 接上舵机,烧 `stage3_servo_lock`,测试开锁/自动回锁。
4. 接上蜂鸣器,烧最终整合版 `stage4_full_core`(别忘了把 `WHITELIST` 也换成你的卡号)。

> 注意:四个工程里的白名单卡号(`WHITELIST`)是各自独立的,每个 `.ino` 文件都要
> 单独改成你自己的卡号,格式必须和阶段1打印出来的一致(小写、无分隔符)。

## 阶段5 —— 智能升级版(`stage5_smart/`)

在阶段4的基础上做了一次"智能升级",接线和阶段4完全一样(**不再需要 DS3231 RTC 模块**,
用WiFi连上网自动对时替代了它)。新增功能:

- **WiFi + NTP 自动对时**:开机连WiFi(最多等10秒),连上就用网络时间给每条事件打时间戳;
  连不上也没关系,门禁核心逻辑(读卡/白名单/开锁)完全不依赖WiFi,照常离线运行。
- **白名单持久化**:白名单存进 ESP32 的 Preferences(NVS闪存),断电不丢,不用每次改代码重烧。
- **主卡录卡**:刷一下"主卡"进入15秒录卡模式(双灯闪烁),这15秒内刷的第一张陌生卡
  会自动加入白名单并存进闪存。主卡本身不会开门,专门用来管理白名单,更安全。
- **防暴力破解**:连续刷5次陌生卡触发30秒锁定,期间红灯常亮、每5秒响一次长报警,
  所有卡(包括白名单卡)一律无效。
- **事件记录 + 网页看板**:内存里保留最近20条刷卡记录,WiFi连上后浏览器访问ESP32的IP
  就能看到当前状态、白名单卡数和记录列表,还有"远程开门"“解除锁定”两个按钮。

**使用前需要改的地方**(文件顶部):
- `WIFI_SSID` / `WIFI_PASS`:改成你家WiFi(改不了也能用,只是没有网页和准确时间)。
- `MASTER_UID`:换成你想当"主卡"的卡号(阶段1刷卡读出来)。
- 白名单不用改代码,首次开机会自动写入占位卡号 `a1b2c3d4`,之后用主卡+新卡的
  方式现场录入你的真实卡号即可(存在闪存里,断电不丢)。

编译验证:`arduino-cli compile --fqbn esp32:esp32:esp32 firmware/stage5_smart`。
